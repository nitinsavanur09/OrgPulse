Section 1
AI Readiness Index
The headline score shown on the cover. A weighted average of the 7 domain scores, rounded to the nearest integer.
OVERALL INDEX src/scoring/engine.ts · scoreFindings()
// Σ (domainScore × domainWeight) across all 7 domains overallIndex = ROUND( data_quality_score × 0.25 // 25% + automation_score × 0.20 // 20% + security_score × 0.15 // 15% + knowledge_score × 0.15 // 15% + metadata_score × 0.10 // 10% + adoption_score × 0.10 // 10% + limits_score × 0.05 // 5% ) // weights must sum to 1.0
50
Hard blocker threshold — domain below this becomes a blocker
+35
Points added to derive target score after remediation
95
Target score cap — never promises perfection
16 wk
Weeks added to today to compute Pilot Ready Date
Target score formula: MIN(overallIndex + 35, 95) — represents where the org lands after completing the remediation roadmap in Section 5.
Section 2
Domain scoring formulas
Each domain starts at 100 and deducts penalty points based on scan signals. All scores are clamped to 0–100. Click a domain to expand the formula.
01
Data Quality & Completeness
25%
›
FORMULAscoreDataQuality()
// Weighted average completion rate (weight = record count per object) avgCompletionRate = Σ(fieldCompletionRate × totalRecords) / Σ(totalRecords) // fieldCompletionRate per field = COUNT(field) / COUNT(Id) × 100 maxDupRate = MAX(duplicateRate across all scanned objects) dupPenalty = maxDupRate > 15% → 30 maxDupRate > 10% → 15 maxDupRate > 5% → 5 otherwise → 0 score = CLAMP(avgCompletionRate − dupPenalty, 0, 100)
Objects with fewer than 100 records are skipped. Duplicate rate = duplicateCount / totalRecords × 100.
02
Automation Health & Conflicts
20%
›
FORMULAscoreAutomation()
faultPathPct = flowsWithNoFaultPath / totalActiveFlows × 100 faultPenalty = faultPathPct > 50% → 40 faultPathPct > 25% → 20 faultPathPct > 10% → 10 otherwise → 0 apexPenalty = apexCoveragePct < 40% → 30 apexCoveragePct < 60% → 15 apexCoveragePct < 75% → 5 otherwise → 0 legacyPenalty = legacyAutomationCount > 10 → 25 legacyAutomationCount > 5 → 10 legacyAutomationCount > 0 → 5 otherwise → 0 score = CLAMP(100 − faultPenalty − apexPenalty − legacyPenalty, 0, 100)
03
Security & Permission Model
15%
›
FORMULAscoreSecurity()
criticalPenalty = criticalCheckCount > 5 → 20 criticalCheckCount > 2 → 10 otherwise → 0 guestPenalty = guestUserRisk = true → 15 | otherwise → 0 score = CLAMP(healthCheckScore − criticalPenalty − guestPenalty, 0, 100)
Source: Salesforce REST endpoint /services/data/v59.0/connect/security/health-check. Returns 404 on Dev Edition — fallback returns 0, which triggers a hard blocker on Dev Edition only. Will return real data on client orgs.
04
Knowledge Base & Grounding
15%
›
FORMULAscoreKnowledge()
// Hard floor — no knowledge base at all IF articleCount = 0 → score = 15 (hard blocker) staleRatio = staleArticleCount / articleCount × 100 gapRatio = coverageGapCount / topCaseReasons.length × 100 stalePenalty = staleRatio > 50% → 40 | >30% → 20 | >15% → 10 | else → 0 gapPenalty = gapRatio > 75% → 40 | >50% → 25 | >25% → 10 | else → 0 score = CLAMP(100 − stalePenalty − gapPenalty, 0, 100) // Volume caps applied after penalty articleCount < 10 → cap score at 30 articleCount < 50 → cap score at 60 articleCount ≥ 50 → no cap
coverageGapCount = number of top case reasons (from Case SOQL scan) with no matching published Knowledge article.

Section 4
Waste calculations (Tier 1 — Verified)
All Tier 1 numbers are derived directly from scan data and confirmed inputs. They are not estimated.
Tier 1 — Verified
Derived from scan data
Inactive user count, login rate, email gap — all from SOQL aggregates. License cost confirmed at interview. No modelling involved.
Tier 2 — Projected
Modelled from scan + benchmarks
Case deflection value uses scan case volume but applies Salesforce published deflection rates and a default cost-per-case. See Section 5.
INACTIVE USER COUNTbuildReportData()
inactiveUserCount = ROUND(licenseCount × (1 − loginRatePct / 100)) // licenseCount — from stakeholder interview (total contracted seats) // loginRatePct — from Domain 6 scan (LoginHistory last 90 days) // Returns 0 if licenseCount not provided at interview
ANNUAL LICENCE WASTEbuildReportData()
licenseWaste = inactiveUserCount × licenseUnitCostMonthly × 12 totalVerifiedWaste = licenseWaste + packageWaste // licenseUnitCostMonthly — default $165/user/month (SF Sales Cloud Professional list price, June 2026) // Override from client contract in client-intake.json → licenseUnitCostMonthly // packageWaste — from interview: AppExchange subscription cost on abandoned packages // Example: 23 inactive × $165/month × 12 = $45,540 licenseWaste
EMAIL GAP & UNREACHABLE CONTACTSbuildReportData()
emailGapPct = ROUND(100 − emailCompletionRate) // Contact object unreachableContactCount = ROUND(contactTotalRecords × emailGapPct / 100) // emailCompletionRate = COUNT(Email) / COUNT(Id) × 100 on Contact
QUARTERLY WASTE (cost-of-inaction section)buildReportData()
quarterlyWaste = ROUND(totalVerifiedWaste / 4) // Used in narrative: "every quarter of inaction costs approximately $X"
Section 5
Agentforce value (Tier 2 — Projected)
These numbers use actual scan case volume but apply published Salesforce deflection benchmarks. Update currentCostPerCase from the client interview.
CASE DEFLECTION VALUEbuildReportData()
monthlyVolume = totalCasesLast3Months / 3 // from scan (COUNT Case last 90 days ÷ 3) || monthlyTransactionVolume // intake override if scan returns 0 deflectionRate = 0.40 // Salesforce State of Service 2025, p.14 deflectedTransactionsPerMonth = ROUND(monthlyVolume × deflectionRate) annualCapacityValueDollars = deflectedTransactionsPerMonth × handlingCostPerTransaction × 12 // handlingCostPerTransaction — from client-intake.json (client interview); null = suppress Section 4 // Example: 2,840/mo × 40% = 1,136 deflected × $18 × 12 = $245,376/yr // Section 4 only renders when handlingCostPerTransaction != null (valueProjection.available = true)
CURRENT ORG EFFECTIVENESS (narrative only)buildReportData()
currentEffectivenessPct = overallIndex ≥ 80 → 70% overallIndex ≥ 60 → 40% otherwise → 15% // Used in narrative text only — not in ROI calculation
Scenario range shown in the Flex Credit model: Conservative 30%, Most Likely 40%, Optimistic 55%. The default ROI slider uses the Most Likely (40%) figure.
Section 6
Flex Credit model
Agentforce uses Flex Credits for consumption-based billing. The model shows what the client pays per case via agent vs. the current human handling cost.
FLEX CREDIT ECONOMICSbuildReportData() · flexCreditScenarios[]
creditsPerTransaction = 4.5 // avg turns per service case conversation creditCostUSD = $0.10 // Salesforce Flex Credit list price flexCreditCostPerTransaction = 4.5 × $0.10 = $0.45 per deflected case // Per scenario (pre-computed in buildReportData, stored in flexCreditScenarios[]): deflectedCount = ROUND(monthlyVolume × scenarioRate) monthlyCreditCost = ROUND(deflectedCount × creditsPerTransaction × creditCostUSD) monthlyNetSaving = ROUND(deflectedCount × (handlingCostPerTransaction − flexCreditCostPerTransaction)) // Example (Conservative, 2,840 cases/mo, $18/case): // deflectedCount = ROUND(2840 × 0.30) = 852 // monthlyCreditCost = ROUND(852 × 4.5 × 0.10) = $383 // monthlyNetSaving = ROUND(852 × (18 − 0.45)) = $14,953

ROI Calculator formulas
Two independent sliders. Tier 1 (verified) is scan-derived and fixed. Tier 2 (projected) is the adjustable Agentforce value estimate.
